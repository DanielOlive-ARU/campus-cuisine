# Ethics and Future Development

## Ethical Considerations

### 1. Data Minimisation
The app should avoid collecting unnecessary personal information. A simple guest checkout flow is enough for the project unless authentication becomes essential.

### 2. User Trust
Prices, quantities, and totals must be accurate and clearly displayed. Users should not be surprised by changes to their order.

### 3. Safe Destructive Actions
Removing items or clearing an order should always require confirmation to avoid accidental data loss.

### 4. Accessibility
The app should use readable font sizes, strong contrast, clear spacing, and touch-friendly controls. This supports inclusive mobile use.

### 5. Security
Even for a student project, backend validation and basic protection of admin endpoints matter. This prevents invalid data and demonstrates professional practice.

### 6. Transparency
If menu items are unavailable or if the backend cannot be reached, the app should give clear feedback rather than failing silently.
