# Restaurant Flyout Menu App – Project Planning Pack

This planning pack is designed for a 2-person full stack university project using **.NET MAUI (frontend)** and **Python FastAPI (backend)**.

## Team
- **Dan** – Backend lead
- **Adam** – Frontend lead

## Project Summary
The application is a cross-platform mobile restaurant ordering app with **flyout navigation**. Users can browse food categories, add items to a shared order, view an order summary, and place an order. The backend provides menu data, item CRUD, and order tracking via REST APIs.

## Architecture Summary
- **Frontend:** .NET MAUI, XAML, C#, MVVM, dependency injection
- **Backend:** Python, FastAPI, SQLite, Pydantic validation
- **Shared Order State:** MAUI singleton/shared service
- **API Contract:** REST + OpenAPI docs generated by FastAPI
- **Images:** dynamically loaded from backend URLs

## Delivery Strategy
1. Implement every **MUST** requirement end-to-end.
2. Add selected **SHOULD** features that are low risk and easy to demonstrate.
3. Only add **MAY** features if the core system is stable.

## Recommended SHOULD Features
- Consistent branding and reusable styles
- Dynamic images from backend
- Offline browsing of cached menu data
- Order cancellation before checkout
- Service/data loading tests
- Order state persistence tests
- GitHub Actions for automated tests

## Out of Scope Unless Finished Early
- Full user authentication flow
- Payment gateway integration
- Analytics dashboard
- Complex admin portal UI

## Suggested Milestones
- **Week 1:** planning, wireframes, API contract, repo structure
- **Week 2:** backend menu endpoints + frontend shell/navigation
- **Week 3:** shared order state + order endpoints + menu pages
- **Week 4:** order summary, place order flow, integration
- **Week 5:** testing, documentation, polish, viva prep
